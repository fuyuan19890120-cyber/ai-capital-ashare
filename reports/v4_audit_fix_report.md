# 策略一 V4 回测审计修复报告

> 2026-07-18 | 分支 `fix-v4-backtest` | 前置:双 agent 对抗性审计;本报告与代码已经过第三只独立 agent 对抗性复审并按其意见修订(v2)

## TL;DR

修复可修的偏差后,**V4(按实际实现=低波+动量组合)的年化为 +12.1%,夏普 0.63**——不是宣称的 +22.9%/0.92;且该数字仍含无法用现有数据修复的幸存者偏差(文献常见 +3~8%/年),真实预期大概率在**个位数年化**。参照:沪深300 买入持有 2016-01(策略首次调仓)~2026-07 年化约 +4.6%、回撤 -45.6%(2015-01 起算则为 +2.3%/-46.7%)。

**旧数字 +22.9%/-28.9%/0.92 无法从仓库任何代码复现**(4 个 worktree 引擎一致;下表 legacy/calendar 两种口径分别只有 +8.9% 和 +12.3%),定稿结果没有可复现出处——这是比任何单个 bug 更需要修的流程问题。

## 必须如实说的五件事

1. **2025 一年贡献了大部分收益**:剔除 2025,其余 10 个建仓年份复合年化仅 **+8.0%**。策略成色高度依赖单一年份,与双低策略"2024-2026 爆炸收益存疑"同款红旗。
2. **幸存者偏差未修**:1141 只全是存活至今的票(零退市股),文献口径美化 +3~8%/年。修复需退市股数据+时点股票池(Baostock `query_all_stock(date)` 可做,已列入后续)。
3. **样本从 2016 起**:warmup 252 天吃掉了 2015(含股灾),"全样本"口径不实。
4. **策略本质是低波+动量**,不是四因子:价值/质量因子在生产代码中不存在(参数从未传入)。ST 股未过滤(`get_st_filters()` 存在但未接线),低波因子理论上偏好 5% 涨跌幅的 ST 票,待定量。
5. **本轮只修了 `stock_backtest.py` 主引擎**:姊妹引擎 `src/risk_backtest.py`(风控对比用)仍保留全部旧 bug(收盘成交/无印花税/停牌回退前收/无涨跌停),后续做风控重评前必须先修它。

## 结论与建议

- 修复后的 V4 相对沪深300 仍有超额(+12.1% vs +4.6%,回撤更浅),**但不是 +22.9% 的"定稿策略"**;扣除幸存者偏差后预期个位数年化,且高度依赖 2025 单年。
- 6/30 实盘 -31.6% 半月亏损的三个成因(未回测宇宙、高贝塔集中违反板块上限、制度月频滞后)中,本轮修复了第一个的度量问题;后两个属于制度/风控设计,是下一步(制度降档改日/周频、风控按尾部指标重评)。
- **流程改进**:任何"定稿"数字必须附带可复现命令与数据快照标识。本报告全部数字可用下述命令复现(数据:data/stocks/ 2026-07-13 快照)。
