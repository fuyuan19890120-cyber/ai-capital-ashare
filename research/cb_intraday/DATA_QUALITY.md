# 可转债分钟数据质量评估报告

> 2026-07-18 | 结论支撑 `research/cb_intraday` 基建设计,探测方法可用本目录代码复现

## TL;DR

1. **Baostock 不支持可转债**(任何频率返回 0 行),分钟数据只能靠 新浪/东财/腾讯 行情 API
2. **没有任何免费源提供超过 1 年的分钟历史** → 分钟档案必须每日采集、滚动积累(存储层已按此设计)
3. 数据质量本身很好:覆盖完整、bar 无缺失、与日线交叉验证一致
4. akshare 的 `bond_zh_hs_cov_min` 封装硬编码 `lmt=66`/`ndays=1`,**不可直接使用**,本基建全部绕过 akshare 直连行情 API

## 各源深度实测(2026-07-17 收盘后)

| 源 | 1分钟 | 5分钟 | 15分钟 | 30分钟 | 60分钟 | 上限机制 |
|---|---|---|---|---|---|---|
| **新浪**(主力) | ✗(最小scale=5) | 21个交易日 | ~3个月 | ~6个月 | **~1年**(2025-06-30起) | 固定1023根 |
| **东财** | 仅当日 241 根(含9:15盘前另有256根) | ~30个交易日(初测) | 未测通 | 未测通 | 未测通 | 服务端窗口 |
| **腾讯** | ~1.3天(320根) | ~6.7天(320根) | - | - | - | 固定320根 |
| **Baostock** | ✗ | ✗ | ✗ | ✗ | ✗ | 不支持转债 |

注:1023 根是**固定 bar 数**上限,低活跃券的窗口会更长(冒烟测试中有券 60 分钟数据回溯到 2023-06)。

## 质量测量(新浪源)

- **覆盖面**:在市 331 只(SH164/SZ164/BJ3),随机抽样 12 只 **12/12 可用**;次新券从上市日起即有数据
- **bar 完整性**:5 分钟 21/22 天恰好 48 根/天(唯一不足的一天是 1023 上限截断的最早一天),**无盘中缺 bar**
- **零成交 bar**:0%
- **交叉验证**:5 分钟聚合收盘价 vs 日线收盘价,最大偏差 0.014 元(舍入级);成交额一致
- ⚠️ **成交量单位陷阱**:新浪分钟 volume 与日线 volume 差 10 倍(手/张口径不一致)。**流动性计算一律用成交额 amount(元)**,基建代码已遵守

## 网络与限频实测

- **东财**:本机直连(IPv4/IPv6)均被服务端断连,需走 SOCKS 代理(`socks5h://127.0.0.1:1081`,可用 `CB_PROXY` 环境变量覆盖);高频请求(约10次/分)后触发风控,冷却期内代理也被断。**已验证 GitHub Actions 美国节点可直连东财**(cb-weekly 项目经验)→ 1 分钟采集器建议部署到 Actions
- **新浪**:未触发限频(0.6-0.9s 间隔,约 1000 次请求),稳定
- **腾讯**:未触发限频,但 320 根上限价值有限

## 采集节奏要求(错过即永久丢失)

| 频率 | 窗口 | 最低采集频率 | 建议 |
|---|---|---|---|
| 1分钟 | 仅当日 | **每个交易日收盘后** | 部署 GitHub Actions(本机东财不稳) |
| 5分钟 | 21个交易日 | 每 2 周 | 每日跟采,成本极低 |
| 15分钟 | ~3个月 | 每月 | 每日跟采 |
| 60分钟 | ~1年 | 每季度 | 每日跟采 |

## 对研究的含义

- **当下可用**:60 分钟 × 1 年 × 全市场,足够做第一轮日内信号粗筛(方向判断、成本敏感性)
- **3-6 个月后**:5/15 分钟档案积累到可支撑精细回测(入场时点、滑点估计)
- **1 分钟/盘口级**:只能从零积累;若某信号在 60 分钟级别都不赚钱,大概率不值得等 1 分钟数据
